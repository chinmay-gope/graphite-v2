# Contributing to Graphite

Thank you for your interest in contributing to Graphite.

---

## Coding Style

- Follow standard Java naming conventions.
- Use meaningful class and method names.
- Prefer immutable result objects.
- Avoid duplicate code.
- Document all public APIs.

---

## Package Structure

Every new feature should belong to an existing package.

Examples:

algorithm/
graph/
builder/
generator/
result/
validation/
util/

Avoid creating miscellaneous utility packages.

---

## Algorithms

Every algorithm should include:

- Time Complexity
- Space Complexity
- Preconditions
- Example Usage

Every algorithm must have:

- Example class
- Benchmark
- Validation

---

## Commit Messages

Use descriptive commit messages.

Examples:

Add Prim's Algorithm

Implement Euler Circuit validation

Refactor GraphBuilder API

Improve RandomGraphGenerator

Avoid generic messages like:

update

changes

fixed

---

## Pull Requests

Every Pull Request should include:

- Description
- Motivation
- Complexity Analysis
- Example Output

---

Happy Coding!
